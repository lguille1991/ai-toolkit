## Philosophy

Accessibility-first selectors. Use the `aria/` selector prefix which queries by ARIA accessible name, mirroring how assistive technologies and users perceive the page.

## Locator Hierarchy (first match wins, top to bottom)

| Priority | Strategy | When to use | Code pattern |
|---|---|---|---|
| 1 | `aria/` selector | Element has an accessible name (from aria-label, text content, or label association) | `$('aria/Submit')` or `$('aria/Email Address')` |
| 2 | `data-testid` attribute | Element has a dedicated test attribute | `$('[data-testid="submit-btn"]')` |
| 3 | `id` selector | Element has a stable, unique `id` | `$('#email-input')` |
| 4 | `name` attribute | Form element with `name` attribute | `$('[name="username"]')` |
| 5 | Text selector (exact) | Element identifiable by exact visible text | `$('=Sign In')` |
| 6 | Text selector (partial) | Element partially matching visible text | `$('*=Sign')` |
| 7 | CSS selector | Minimal CSS when no semantic attr available | `$('button.submit-btn')` |
| 8 | XPath | **Fallback only** | `$('//div[@class="form"]/button')` |

## Framework-Specific Rules

- The `aria/` selector is WebdriverIO's most recommended approach — it uses the browser's accessibility tree.
- Prefer `aria/` over text selectors (`=`) because `aria/` also matches aria-label and label associations, not just visible text.
- Chain selectors for specificity: `$('form').$('aria/Submit')`.
- Use deep selectors for Shadow DOM: `$('>>>.shadow-element')`.
- Avoid long selector chains — keep them to 2 levels max.
- For class-based selectors, prefer attribute selectors over `.class` syntax.

# Test Case Standards — Section Definitions

## Sections

### std — Shared Standards
Impact: HIGH
Order: 1
Rules that apply to every test case produced, evaluated, audited, or normalized.

### gen — Generation Rules
Impact: HIGH
Order: 2
Rules specific to Mode A (Generate) — writing new test cases from features or stories.

### eval — Evaluation Rules
Impact: MEDIUM
Order: 3
Rules specific to Mode B (Evaluate) — scoring and reviewing existing test cases.

### audit — Audit Rules
Impact: MEDIUM
Order: 4
Rules specific to Mode C (Audit) — analyzing full test suites for coverage gaps.

### norm — Normalization Rules
Impact: MEDIUM
Order: 5
Rules specific to Mode D (Normalize) — converting legacy test cases to team standard.

### ref — Reference Rules
Impact: HIGH
Order: 6
Reference lookup tables and technical definitions loaded during skill execution.

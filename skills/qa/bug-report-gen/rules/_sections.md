# Bug Report Standards — Section Definitions

## Sections

### std — Shared Standards
Impact: HIGH
Order: 1
Rules that apply to every bug report produced, evaluated, enriched, or normalized.

### draft — Drafting Rules
Impact: HIGH
Order: 2
Rules specific to Mode A (Draft) — writing new bug reports from raw descriptions.

### eval — Evaluation Rules
Impact: MEDIUM
Order: 3
Rules specific to Mode B (Evaluate) — scoring and reviewing existing bug reports.

### enrich — Enrichment Rules
Impact: MEDIUM
Order: 4
Rules specific to Mode C (Enrich) — expanding minimal defect stubs.

### norm — Normalization Rules
Impact: MEDIUM
Order: 5
Rules specific to Mode D (Normalize) — converting legacy reports to team standard.
